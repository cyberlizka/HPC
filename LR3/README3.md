# Bilateral (CPU/GPU с текстурной памятью CUDA)

## Описание работы программы

Программа реализует **Bilateral** полутонового изображения с использованием **9-точечного окна (3×3)**. 
**CPU** (последовательная обработка на центральном процессоре)  
**GPU** (параллельная обработка на видеокарте с применением **текстурной памяти CUDA**)  

## Основные этапы работы

### Загрузка входного изображения
- Считывается файл `boat.bmp` (grayscale, 8 бит на пиксель) с помощью OpenCV.
- Исходное изображение преобразуется в тип `float32` для точных вычислений.

### Параметры фильтра
- **sigma_spatial (σ_d)** – пространственный параметр, определяющий радиус сглаживания по расстоянию.
- **sigma_range (σ_r)** – диапазонный параметр, определяющий чувствительность к разнице яркостей.

### CPU‑фильтрация
- Изображение дополняется по краям методом `edge` (копирование ближайших пикселей) для корректной обработки границ.
- Для каждого пикселя `(i, j)`:
  - Вырезается окно 3×3.
  - Вычисляется нормировочная константа `k` как сумма произведений пространственного и диапазонного весов.
  - Вычисляется взвешенное среднее интенсивностей в окне с весами `(пространственный вес × диапазонный вес)`.
  - Результат делится на `k` (или остаётся равным центральному пикселю, если `k` близко к нулю).
- Замер времени выполняется через `perf_counter()`.

### GPU‑фильтрация с текстурной памятью
- Изображение копируется на устройство (GPU) после дополнения краёв.

#### Создание текстуры CUDA:
- Описывается формат канала (`float32`).
- Создаётся `CUDAarray` и в него копируются данные.
- Определяется ресурсный и текстурный дескрипторы (`ResourceDescriptor`, `TextureDescriptor`).
- Режим адресации – `clamp` (вне границ – ближайший пиксель), фильтрация – линейная.

#### Распараллеливание на GPU
- Ядро `bilateralKernel` запускается с блоками 16×16 потоков.
- Каждый поток обрабатывает один пиксель: выбирает 9 соседей через текстуру, вычисляет нормировку и новое значение.
- Измерение времени GPU – с помощью событий CUDA (`cp.cuda.Event`).
- Результат копируется обратно в оперативную память (`cp.asnumpy`).

### Сохранение результатов
Отфильтрованные изображения (GPU и CPU) приводятся к диапазону `[0, 255]` и типу `uint8`.
Записываются в файлы BMP: `output_gpu.bmp` и `output_cpu.bmp`.

### Результат визуализации позволяет наглядно убедиться:
- в сохранении контуров (края лодки, мачты, отражений не размываются);
- в сглаживании шумов на однородных участках (вода, небо);
- в идентичности выходов CPU и GPU (различия могут быть в пределах вычислительной погрешности).
