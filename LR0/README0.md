# Лабораторная работа: Умножение матриц (CPU vs GPU)

## Описание реализации
- **Все эксперименты проводились в среде Google Colab с использованием графического ускорителя NVIDIA Tesla T4**
### Алгоритм умножения матриц
Классический алгоритм: C[i][j] = Σₖ A[i][k] * B[k][j].

### Распараллеливание на GPU
- **CUDA‑ядро**: каждый поток вычисляет один элемент результирующей матрицы.
- **Сетка и блоки**: сетка двухмерная, размер блока 16×16 (256 потоков). Блоки покрывают всю область результирующей матрицы.
- **Память**: данные копируются в глобальную память GPU, результат возвращается.

### Почему именно так?
- Двумерная сетка удобна для сопоставления с индексами матрицы.
- Размер блока 16×16 выбран как компромисс между количеством потоков и загрузкой ядер, обеспечивая оптимальное использование вычислительных ресурсов.
- Наивная реализация (без разделяемой памяти) упрощает понимание, но при больших размерах уступает оптимизированным библиотекам.

## Анализ результатов

### Ключевые результаты
- **Максимальное ускорение**: 42 590 раз для матриц 2000×2000
- **Время на GPU для 2000×2000**: 0,0793 секунды против 3378,66 секунд на CPU
- **Масштабирование CPU**: кубическая зависимость O(N³), что подтверждается ростом времени с 0,404 с (100×100) до 3378,66 с (2000×2000)
- **Масштабирование GPU**: рост времени значительно медленнее теоретического O(N³) благодаря эффективной параллелизации при увеличении размера задачи

## Выводы

1. **GPU обеспечивает кардинальный прирост производительности** уже начиная с матриц 250×250, где ускорение превышает 5000 раз.

2. **Эффективность параллелизации растет с увеличением размера** задачи — чем больше матрицы, тем более полно загружаются вычислительные ресурсы GPU. Ускорение увеличивается с 335 раз для 100×100 до 42 590 раз для 2000×2000.

3. **Практическая значимость**: задача, требующая на CPU почти часа вычислений (2000×2000 — 56 минут), выполняется на GPU менее чем за 0,1 секунды.

4. **Подтверждение теоретических ожиданий**: полученные зависимости времени выполнения и ускорения соответствуют теоретическим моделям параллельных вычислений и демонстрируют эффективность выбранной архитектуры (блоки 16×16, прямое отображение элементов на потоки).

5. **Масштабируемость**: предложенная конфигурация блоков (16×16) позволяет эффективно загружать до 256 потоков на блок, а общее количество блоков растет пропорционально площади результирующей матрицы, обеспечивая хорошую масштабируемость.
